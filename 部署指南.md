# 漆彩部署指南 → paint.zengzhisui.com

跟 [calc.zengzhisui.com](https://calc.zengzhisui.com) 同一套流程(GitHub Pages + Cloudflare DNS)。預估 15 分鐘。

## 前置:本專案已備妥
- ✅ Git repo 已 init,已有 commit
- ✅ `CNAME` 檔案已含 `paint.zengzhisui.com`
- ✅ 純靜態,無需 build

## 步驟 1:建 GitHub repo 並推上去

在 GitHub 網頁建一個新 repo(例如 `qicai-paint`,設為 Public 才能用免費 GitHub Pages)。然後在專案目錄執行:

```bash
cd "漆彩專案目錄"
git remote add origin https://github.com/<你的帳號>/qicai-paint.git
git push -u origin main
```

> 若沒裝 `gh` CLI,直接用上面的 git 指令即可;認證會跳瀏覽器或要求 Personal Access Token。

## 步驟 2:啟用 GitHub Pages

GitHub repo → **Settings** → **Pages**：
- Source 選 **Deploy from a branch**
- Branch 選 **main** / **/(root)**
- 儲存

幾分鐘後 GitHub 會顯示 `Your site is live at https://<帳號>.github.io/qicai-paint/`。
因為有 `CNAME` 檔案,Pages 會自動設定自訂網域為 `paint.zengzhisui.com`。

## 步驟 3:Cloudflare 設 DNS

到 Cloudflare → zengzhisui.com 網域 → **DNS** → **Add record**：

| 欄位 | 值 |
|---|---|
| Type | `CNAME` |
| Name | `paint` |
| Target | `<你的帳號>.github.io` |
| Proxy status | 🟠 Proxied(或先 DNS only 確認 SSL 後再開 proxy) |

> 注意:Target 是 `<帳號>.github.io`(**不含** repo 路徑)。

## 步驟 4:等 HTTPS 憑證

GitHub Pages 會自動為 `paint.zengzhisui.com` 申請 Let's Encrypt 憑證(5-30 分鐘)。
在 Settings → Pages 勾選 **Enforce HTTPS**。

## 更新網站

之後改動只要:
```bash
git add -A && git commit -m "更新內容" && git push
```
GitHub Pages 約 1 分鐘自動重新部署。
