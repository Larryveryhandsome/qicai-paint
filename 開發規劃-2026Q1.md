## 1. 用戶與痛點(2026.01.21 重新定位)

YuXun 2026.01.21 18:54 明確澄清:
> 「這東西不是給設計師看的,是給業主跟廠商看的,不然怎麼知道要買哪一個」

| 用戶 | 痛點 | 漆彩解法 |
|---|---|---|
| **業主** | 跟設計師討論顏色時,設計師指定 A 牌,但業主常找到只賣 B 牌的油漆行,不知道對應色號 | 輸入 A 牌色號 → 即得 B 牌相近色號 |
| **承包商 / 油漆師傅** | 客戶丟一個某廠牌色號,但手邊只有別家庫存,需快速判斷可不可以替代 | 同上,加上「相似度」評分 |
| **DIY 裝修者** | 看 IG/Pinterest 想要某色,但不知道台灣買得到的對應商品 | 輸入 HEX/RGB → 推薦最接近的台灣廠牌色號 |

**非目標用戶:**
- 專業設計師(他們有色卡)
- 油漆品牌行銷(這不是他們的工具)

## 4. 資料 Schema

### `colors.json`(主資料庫)
```json
{
  "schema_version": "2.0",
  "updated_at": "2026-05-27",
  "colors": [
    {
      "brand": "虹牌",
      "code": "1801",
      "name": "純白",
      "hex": "#F5F5F0",
      "rgb": [245, 245, 240],
      "lab": [96.2, -0.5, 2.1],
      "cmyk": [0, 0, 2, 4],
      "source": "colortell.com",
      "source_url": "https://www.colortell.com/colorbook/?id=...",
      "reference_price_5l": null
    }
  ]
}
```

### `brands.json`(廠牌清單,供下拉選單動態載入)
```json
[
  {"id": "rainbow", "name": "虹牌", "official_url": "https://www.rainbowpaint.com.tw"},
  {"id": "nippon", "name": "立邦", "official_url": "https://www.nipponpaint.com.tw"},
  {"id": "dulux", "name": "得利", "official_url": "https://www.dulux.com.tw"},
  {"id": "qingye", "name": "青葉", "official_url": "..."},
  {"id": "toa", "name": "TOA", "official_url": "..."}
]
```

**正規化規則:**
- `hex` 全部大寫 `#FFFFFF`
- `rgb` 為 `[0-255, 0-255, 0-255]`
- `lab` 為 `[0-100, -128~127, -128~127]`
- 缺 LAB 的紀錄,用 RGB→XYZ→LAB 補齊(後處理 script)

## 5. Phase 1 — MVP 重啟(2-3 週)

### 5.1 資料層(D 系列任務)

#### Task 1 — 爬 colortell.com
- **目標廠牌**:虹牌 / 立邦 / 得利 / 青葉 / TOA
- **目標筆數**:每廠牌 200-500 筆,合計 1000-2500
- **技術**:Python + requests + BeautifulSoup
- **節流**:每 request 間隔 ≥ 1 秒,避免被擋
- **法律**:加 `User-Agent`,輸出資料保留 `source_url` 欄位以便註明來源
- **輸出**:`raw/colortell-{brand}.json`
- **驗收**:每廠牌至少 200 筆,HEX 與 RGB 對得起來

#### Task 2 — 爬 qtccolor.com
- 補強 colortell 蓋不到的廠牌與色號
- HTML 結構不同,需另寫 parser
- **輸出**:`raw/qtccolor-{brand}.json`

#### Task 3 — 正規化與 LAB 補齊
- 合併 raw 資料,以 `{brand}-{code}` 為主鍵去重
- 計算 LAB(若爬下來只有 RGB)、CMYK
- **輸出**:`colors.json` 與 `brands.json`,放在專案根目錄,取代 `sample-data.json`

### 5.2 UI 層(U 系列任務)

#### Task 4 — 文案與定位改寫
- **Hero 標題**:「跨品牌油漆色彩比對平台」 → 「業主與承包商的色號對照工具」
- **Hero 副標**:強調「拿著色號,找到任何品牌的相近色」
- **About 區塊**:刪除「電商導購」、把「給設計師」改成「給業主與承包商」
- **Footer**:拿掉假聯絡資訊,改為留白事務所基本資訊或留空

#### Task 5 — 比對結果加 RGB/CMYK
- 修改 [script.js](script.js) 的 `displayResults()`
- 每張結果卡片下方新增兩行:
  ```
  RGB: 245, 245, 240
  CMYK: 0, 0, 2, 4
  ```

#### Task 6 — 動態載入廠牌下拉選單
- 把 [index.html](index.html) 第 70-77 行的 `<select>` 寫死選項改成空,改在 JS 從 `brands.json` 動態 populate
- 好處:之後新增廠牌不用改 HTML

### 5.3 部署層(B 系列任務)

#### Task 7 — 部署到 paint.zengzhisui.com
1. 在 GitHub 開新 repo `paint-color-match`(或在現有 lvit-web repo 開 `/paint` 子路徑)
2. 把改好的檔案推上去
3. 啟用 GitHub Pages
4. Cloudflare 新增 CNAME:`paint.zengzhisui.com → lvit-studio.github.io`
5. 等 HTTPS 自動 issue(5-10 分鐘)
6. 驗證:用手機開 https://paint.zengzhisui.com,色號查詢能跑通

#### Task 8 — 解封專案目錄
- 目前路徑:`D:\...\封存專案\漆彩-封存網頁\`
- 建議改為:`D:\...\啟用中專案\漆彩\`(實際命名待您決定)
- 或保持原路徑、僅重命名為「漆彩」去掉「封存」二字

## 6. Phase 2 — 商業化強化(1-2 週,Phase 1 上線後再啟動)

### Task 9 — 各廠牌大略市價
- 在 `colors.json` 新增 `reference_price_1l` / `reference_price_5l` 欄位
- 來源:PChome / momo / 蝦皮搜尋該廠牌的常見售價,人工填入
- 顯示格式:「5L 約 NT$1,200-1,500」
- 注意:廠牌應加註「參考價,實際以店家為準」

### Task 10 — 附近油漆行(Google Maps)
- 新增「附近油漆行」區塊
- 使用 Google Maps Embed API(免費,不用 API key)
- 搜尋字串:`油漆 + ${user_city}`
- Geolocation 取使用者位置(需授權),fallback 為手動選縣市

### Task 11 — 廣告版位
- 比對結果上下方各預留一條 banner(728x90 / 320x100 RWD)
- 暫時放 placeholder,實際合作對象之後再談
- 預留 `<div class="ad-slot" data-slot="result-top"></div>` 方便日後接 AdSense

### Task 12 — 收藏與比對歷史
- localStorage 存最近 10 筆比對 + 收藏清單
- 首頁右上「我的收藏」入口
- 無需後端

## 8. 風險與緩解

| 風險 | 機率 | 影響 | 緩解 |
|---|---|---|---|
| colortell.com 反爬蟲 | 中 | 高 | 節流、User-Agent rotation、改 Playwright |
| 爬蟲版權爭議 | 低 | 中 | 標註來源、僅做相似色推薦不直接複製大量色卡圖、聲明非商業用途 |
| 色號與品牌官方對不上(爬蟲資料有誤) | 中 | 高 | 上線前抽樣 20 筆人工核對;留「回報錯誤」按鈕 |
| LAB 數值計算誤差 | 低 | 中 | 用 [chroma.js](https://gka.github.io/chroma.js/) 函式庫而非自寫 |
| 手機顯示色 vs 實際油漆色差(2024.10.07 旭討論過) | 高 | 中 | 結果頁明顯位置加 disclaimer:「螢幕色彩僅供參考,實際以色卡為準」 |
| Phase 2 廣告/市價拖到沒人推進 | 中 | 低 | Phase 1 上線後立刻找品牌或油漆行談合作 |
